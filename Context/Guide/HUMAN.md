# Guide

_Plain-language user guides: how each part of the system works and how to get the best out of it._

## What this folder is for

One page per mechanism, written for you (not the AI). Each answers the same five questions: what it's for, how it works, when it touches you, best practices, and where to dig deeper. When you want to understand a mechanism rather than run it, start here. For a one-word definition, use the [[glossary]]. For the exact procedure the AI follows, the guide page links to its `SKILL.md` or system file.

This is the ongoing reference. The owner's manual (`HUMAN.md` in the vault root) is the one-page overview and getting-started path; the Guide is where you look things up after that.

Every page follows `_template.md`. New pages get created as mechanisms ship; see the coverage check in the weekly health check.
