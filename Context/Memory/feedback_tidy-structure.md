---
name: feedback_tidy-structure
description: Keep the vault tidy — every file lives in a folder that makes sense to a human, no stray files at folder tops
type: feedback
---

Don't leave loose files sitting at the top of a folder. Every file should live somewhere a human can immediately understand why it's there. The vault is shared — structure has to make sense to the user, not just to me.

**Why:** A single open-work.md dropped directly in `Context/` was confusing — the user couldn't tell which folder it belonged to or why it was hanging out on its own.

**How to apply:** When creating a file, place it in the folder with its natural siblings (e.g. open-work.md lives in `History/` with session notes — they're updated together and form the full session picture). If no folder fits, ask before creating a stray file or a new single-file folder.
