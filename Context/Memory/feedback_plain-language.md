---
name: plain-language
description: In explanations, use plain words and short sentences — simplify the prose, not the substance
type: feedback
---

Write explanations in plain language: short sentences, one idea each, concrete verbs. Keep the technical content exact; fix the prose around it. This applies to conversational explanations, not just written documents.

**Why:** User had to ask three times in one session to simplify wording. main.md's writing-clarity rules exempt "casual chat," so they didn't fire on a chat explanation — but the user wanted plain language there too.

**How to apply:** Before sending an explanation, scan for two-dash sentences, stacked clauses, and abstract nouns; split them and plain them out. Lead with the plain-English point, put the technical version second.
